Implementation Checklist

Six things that must ship.

  1. 01

    Viewport Meta Tag

    width=device-width, initial-scale=1 — the key to the kingdom. Without it, mobile browsers render at 980px virtual width.

  2. 02

    Mobile-First Base CSS

    Write all base styles for the smallest screen first. Enhance upward with @media (min-width: …).

  3. 03

    Grid (Macro) + Flexbox (Micro)

    CSS Grid places major page sections. Flexbox aligns items inside each component. Right tool, right job.

  4. 04

    Fluid Units

    %, rem, vw, and clamp() everywhere. No fixed pixel widths on layout containers.

  5. 05

    Hamburger / Popover Navigation

    Desktop nav collapses to a hamburger on mobile. Uses the HTML Popover API — zero JavaScript required.

  6. 06

    Accessible Touch Targets & Zoom

    All interactive elements are at least 44×44px. Never disable user zoom — allow up to 500% magnification.

Core Concepts

The techniques.

Strategy

Mobile-First

Base CSS targets the smallest viewport. Complexity increases with space. The pyramid: Mobile Core → Tablet → Desktop (High-Res).

Layout

Grid + Flexbox Harmony

Grid is the house — macro page structure. Flexbox is the furniture — micro component alignment. Use auto-fit + minmax() for intrinsically responsive grids.

Typography

Fluid Typography

Use clamp(1rem, 2.5vw, 2rem) for smooth, continuous font scaling — no sudden jumps at breakpoints. Modern over traditional.

Touch UX

Designing for Thumbs

Every interactive element meets the 44×44px minimum touch target size. The finger is less precise than a mouse — design for it first.

Breakpoints

Bridges, Not Fences

Design for content breakage, not device models. Break at 480px, 768px, 1024px — wherever your layout starts to struggle.

A11Y

Accessibility is Non-Negotiable

Allow up to 500% zoom. Never use user-scalable=no. WCAG compliance means every person on every device can use your product.
